# 控制台面板重构总任务清单

## 文档说明

本文档用于统一管理“新控制台面板”项目的全量任务范围。

目标：

- 新建一套独立于旧 `monitor data-management` 的面板体系
- 第一阶段只实现 JDBC 客户端
- 后续扩展 Redis 等客户端
- 旧数据管理相关代码在新体系可替代后删除

更新规则：

- `状态` 只允许使用：`未开始`、`进行中`、`已完成`、`已取消`
- 每完成一个子项时，同步更新本文件和《控制台面板重构分步任务清单》
- 如范围变化，优先更新本文件，再更新分步清单

## 当前范围

### 涉及仓库

1. `spring-support-parent-starter`
2. `spring-support-api-starter`
3. `vue-support-parent-starter`

### 暂不处理

1. `utils-support-parent-starter`
2. Redis 客户端具体实现
3. 在线 DDL 修改、索引修改提交
4. 旧数据管理逻辑的功能兼容迁移

## 总目标

### 产品目标

1. 新建独立控制台面板
2. 支持 JDBC 数据源接入
3. 支持数据库/表浏览
4. 支持表结构查看
5. 支持类 Navicat 的搜索能力
6. 支持 SQL 在线调试
7. 支持文档查看、文档生成
8. 支持 AI 辅助解释与生成

### 技术目标

1. 前端使用独立 `pages` 页面实现，不复用旧数据管理页面
2. Spring 端新增独立面板模块，不复用旧 monitor 数据管理业务实现
3. API 端剥离旧 monitor 数据管理接口
4. 保留 `job-starter` 与 `spring-support-sync-data-starter` 作为后台增强能力
5. 面板连接具备缓存能力

## 风险清单

### 高风险

1. 删除旧 monitor 代码后出现隐性编译依赖
2. 前端菜单/路由残留导致构建失败
3. API 端 monitor 相关服务耦合过深

### 中风险

1. JDBC 元数据接口抽象不合理导致后续 Redis 扩展返工
2. AI 入口过早耦合业务页面
3. job/sync-data 接入过早导致第一阶段复杂度上升

### 低风险

1. 文档更新遗漏
2. 路由命名不统一
