## Docker 扩展分支规划（已补全执行版，2026-04-15）

### 目标定位

1. 新增 `vue-support-parent-starter/pages/docker`（`@pages/docker`）作为“服务器维度 Docker 运维页”，交互模型对齐 `@pages/soft`。
2. 将 Docker 能力作为 `pages/server` 的扩展入口，不单独割裂业务上下文。
3. 完成 Docker 检测/安装、容器/镜像/Compose 管理、任务中心与实时日志闭环。

### 现状基线（已核验代码）

1. 前端已有 Docker 相关沉淀：`apps/vue-support-monitor-starter/src/api/docker/*` 与 `apps/vue-support-monitor-starter/src/views/docker/*`，可复用 API 归一化与页面交互模型。
2. 后端已有基础能力：`ServerCommandExecutorSpi`（`LOCAL/SSH/WINRM`）、`ServerRealtimePublisher`、`ServerRealtimeEnvelope`、`ServerSocketEvents`。
3. `pages/server` 已有任务中心与多类实时流（metrics/process/service/ai），Docker 可以按同一流式框架接入。

### 范围冻结（一期）

1. In Scope：
   - Docker 可用性检测 + 安装向导（任务化）
   - 容器列表/详情/启停/重启/删除/日志
   - 镜像列表/拉取/删除
   - Compose 文件读取/保存/执行（`up/down/restart/pull`）
   - Docker 任务中心（进度、阶段、日志、失败原因）
2. Out of Scope（一期不做）：
   - Swarm/K8s 编排
   - 多主机批量编排
   - 镜像仓库高级策略（镜像签名、策略审计）

### 信息架构（前端）

1. `@pages/docker` 主视图固定 tab：
   - `docker-home`：容器概览（不可关闭）
   - `docker-install`：安装向导（可关闭）
   - `docker-compose-editor`：Compose 编辑与执行（可关闭）
   - `docker-task-center`：任务中心（可关闭）
2. `pages/server` 集成策略：
   - 在 `ServerHostPage` 的服务器动作区新增“Docker”入口
   - 点击后携带 `serverId` 打开 `@pages/docker`
   - `@pages/server` 与 `@pages/docker` 状态隔离，仅通过 `serverId` 关联

### 后端接口契约（冻结版 v1）

1. 统一前缀：`/server/docker/*`，落点模块：`spring-support-server-starter`
2. 检测与安装：
   - `GET /server/docker/capabilities?serverId=`
   - `POST /server/docker/install`
3. 容器：
   - `GET /server/docker/containers?serverId=&keyword=&status=`
   - `GET /server/docker/containers/{containerId}?serverId=`
   - `POST /server/docker/containers/{containerId}/start|stop|restart|remove`
   - `GET /server/docker/containers/{containerId}/logs?serverId=&lines=`
4. 镜像：
   - `GET /server/docker/images?serverId=&keyword=`
   - `POST /server/docker/images/pull`
   - `POST /server/docker/images/{imageId}/remove`
5. Compose：
   - `GET /server/docker/compose?serverId=&path=`
   - `PUT /server/docker/compose`
   - `POST /server/docker/compose/up|down|restart|pull`
6. 任务：
   - `GET /server/docker/tasks/{taskId}`
   - `GET /server/docker/tasks/{taskId}/logs`

### 任务与事件契约（冻结版 v1）

1. `ServerSocketEvents` 新增：
   - `SERVER_DOCKER_TASK`
   - `SERVER_DOCKER_LOG`
   - `SERVER_DOCKER_STATS`
2. 事件 envelope 继续使用 `ServerRealtimeEnvelope`，`data` 最小字段：
   - `taskId`、`serverId`、`eventType`、`stage`、`progress`、`message`、`payload`、`timestamp`
