# Server AI 集成回归清单

用于覆盖任务 `16/21/26` 的联调检查。

联调完成后，把结果回填到 [server-ai-integration-report-template.md](./server-ai-integration-report-template.md)。

## 1. AI 激活状态

- 打开服务器页基础信息
- 校验 `AI 能力` 显示已激活或明确失败原因
- 未配置 provider 时，应显示具体原因而不是空白

## 2. 服务器稳定性分析

- 在预警中心点击 AI 图标
- 校验生成当前时间点的稳定性结论
- 校验原因、建议、provider 信息都可见

## 3. 服务启动失败分析

- 人为制造启动失败
- 点击启动
- 校验返回“AI 诊断中”或失败后补诊断
- 打开服务日志，确认以下字段可见:
  - `aiReason`
  - `aiSolution`
  - `knowledgeId`
  - `expireAt`

## 4. AI 修复启动

- 在最近一次失败服务上点击 `AI 修复启动`
- 校验生成一条新的服务操作日志
- 校验修复脚本和结果输出都被记录

## 5. 项目管理页

- 打开项目管理页
- 触发启动/停止/重启/状态检查
- 校验实时进度可通过 soft socket 更新
- 校验日志追尾可看到新增输出
- 校验 AI 失败诊断区能读取最新服务失败分析

## 6. 进程管理

- 打开进程管理
- 对指定进程执行 AI 分析
- 校验风险等级和建议展示

## 7. 知识库沉淀

- 对相同失败场景重复触发
- 校验 `server_service_ai_knowledge` 是否按知识 key 合并
- 校验前端能看到最新知识库编号

## 8. 回归结论

联调通过前，不要把 `25/26` 标为完成项。当前仓库已补展示、入口与记录链路，真实环境验证仍需在测试服务器执行。
