# 支付能力重构计划清单

更新时间：2026-03-19

## 1. 当前现状结论

- `spring-support-payment-starter` 目前同时存在两套支付模型：
  - 新模型：`merchant` / `merchant_channel` / `payment_order` / `transaction_record`
  - 遗留模型：`pay_merchant_*`
- 后端文档、数据库表、Java 实体、Controller、前端 API 契约彼此不一致，当前状态不适合直接继续堆功能。
- `vue-support-payment-starter` 当前是演示型页面，接口路径仍然指向 `/api/payment/*`，与现有后端 `/api/merchant`、`/api/channel`、`/api/order` 不匹配。
- 状态机代码存在基础结构，但订单字段、状态更新、退款流和流水记录还没有形成闭环。
- 当前重构目标不是一次性打通全部第三方真实支付 SDK，而是先把“商户管理 + 支付方式配置 + 开通引导 + 订单生命周期 + 前端管理台”做成一致、可扩展、可继续接 SDK 的基线版本。

## 2. 目标范围

### 2.1 后端目标

- 支持商户增删改查、激活、停用、基础设置维护。
- 支持商户开通支付方式：
  - 微信支付：小程序、JSAPI、H5、APP、Native
  - 支付宝：电脑网站、手机网站、APP、当面付、沙箱
  - 综合支付：用于聚合平台或内部统一路由
  - 钱包：站内余额、充值、扣款、退款
- 支持支付方式配置加密存储、启停控制、开通状态维护、开通指引查询。
- 支持订单创建、删除、支付发起、支付成功、支付失败、完成、退款申请、退款成功、退款失败、超时取消。
- 支持订单状态机和状态流转日志。
- 支持交易流水管理。
- 支持分布式部署，多实例下订单创建、状态流转、超时扫描都要具备幂等和并发安全。
- 支持给前端返回“开通流程弹窗”所需的官方链接、步骤和资料清单。

### 2.2 前端目标

- 支持商户管理。
- 支持商户下支付方式管理。
- 支持开通指引弹窗/抽屉。
- 支持订单管理。
- 支持流水管理。
- 支持按商户、渠道、状态搜索。
- 支持订单状态操作按钮和状态流转展示。

## 3. 本次重构原则

- 统一使用新模型，不再继续扩展遗留 `pay_merchant_*` 设计。
- 所有前端接口契约以重构后的后端接口为准。
- 先做“管理台可用 + 领域模型完整 + 扩展点清晰”，再逐步接第三方真实交易链路。
- 敏感字段必须加密存储，前端返回掩码，不直接回传明文密钥。
- 订单创建、支付回写、退款回写、超时关单必须按多节点并发场景设计，不能依赖单机内存状态。
- 支付方式开通指引必须区分：
  - 官方平台自行开通
  - 需要商户平台补充产品权限
  - 内部配置即可启用

## 7. 支付方式开通与弹窗指引

以下链接与说明基于 2026-03-19 检索到的官方站点信息整理，前端弹窗应内置“官方入口 + 资料清单 + 后台配置项说明”。

### 7.1 微信支付

#### 微信支付商户入驻与基础接入

- 官方接入入口：`https://pay.wechatpay.cn/`
- 官方小程序支付接入指引：`https://pay.wechatpay.cn/static/applyment_guide/applyment_detail_miniapp.shtml`
- H5 支付权限申请文档：`https://pay.wechatpay.cn/doc/v3/merchant/4012791841`
- H5 支付开发指引：`https://pay.wechatpay.cn/doc/v3/merchant/4012791831`
- H5 下单文档：`https://pay.wechatpay.cn/doc/v3/merchant/4012791834`
- APIv3 公钥/平台证书说明：`https://pay.wechatpay.cn/doc/v3/partner/4015419376`

#### 微信支付开通要点

- [ ] 先完成微信支付商户号申请。
- [ ] 将商户号与对应 AppID 绑定。
- [ ] 在商户平台配置 APIv3 密钥。
- [ ] 根据证书模式下载并配置平台证书或微信支付公钥。
- [ ] 补充支付回调地址。
- [ ] 对于 H5 支付，还要在商户平台提交支付域名、ICP备案截图和经营场所说明。
- [ ] 对于小程序 / JSAPI，前端必须具备合法 AppID / OpenID 获取链路。

#### 微信支付弹窗内容要求

- [ ] 展示“去微信支付商户平台开通”按钮。
- [ ] 显示当前需要的资料：营业执照、法人信息、结算账户、AppID、域名、备案截图。
- [ ] 显示系统内要填写的字段：`appId`、`mchId`、`apiKeyV3`、`privateKeyPath`、`notifyUrl`。
- [ ] 提示 H5/小程序能力不是创建商户后自动拥有，需要单独申请产品权限。

### 7.2 支付宝

#### 支付宝开放平台与沙箱

- 官方开放平台：`https://open.alipay.com/`
- 官方产品文档入口：`https://open.alipay.com/productDocument.htm`
- 官方支持中心（含沙箱调试入口）：`https://open.alipay.com/support/supportCenter.htm`
- 官方服务商接入与当面付介绍：`https://open.alipay.com/paymentServicer/paymentServicer.htm`
- 官方沙箱手册 PDF：`https://os.alipayobjects.com/rmsportal/qzepRfDMREmzZQHMwXai.pdf`

#### 支付宝开通要点

- [ ] 注册并完成企业支付宝账号认证。
- [ ] 在开放平台创建应用。
- [ ] 为应用申请具体产品能力：电脑网站支付、手机网站支付、APP 支付、当面付等。
- [ ] 配置应用公私钥与异步通知地址。
- [ ] 联调阶段使用官方沙箱环境，正式环境切换正式网关。
- [ ] 若走服务商代商户签约模式，需额外完成商户签约、授权与审核流程。

#### 支付宝弹窗内容要求

- [ ] 展示“去支付宝开放平台”按钮。
- [ ] 展示“去沙箱调试”按钮。
- [ ] 显示需要准备的资料：企业账号、应用信息、网站域名或 App 信息、公私钥、回调地址。
- [ ] 对当面付说明“如走服务商模式，代签约后仍需商户授权”。

### 7.3 综合支付

- 综合支付不是固定第三方官方产品，属于系统路由层能力。
- 弹窗应说明：
  - [ ] 当前配置对应哪一个聚合服务商或内部网关
  - [ ] 是否需要线下签约
  - [ ] 是否依赖微信/支付宝已先行开通
  - [ ] 系统内的路由标识、签名密钥、回调地址配置项

### 7.4 钱包

- 钱包属于站内能力，不依赖第三方官方开户。
- 弹窗应说明：
  - [ ] 是否启用钱包余额账户
  - [ ] 是否允许充值
  - [ ] 是否允许退款原路退回钱包
  - [ ] 风控和对账责任归平台自身承担
