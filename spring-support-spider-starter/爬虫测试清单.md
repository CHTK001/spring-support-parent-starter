# 爬虫测试清单

## 1. 测试基线

本清单完全对应 [`爬虫设计文档.md`](./爬虫设计文档.md) 与 [`爬虫任务清单.md`](./爬虫任务清单.md) 的最新基线。

以下旧结论全部失效：

- 旧版“左列主页 + 右列工作台”的 UI 烟测结论。
- 旧版 `SpiderWorkbenchPage.vue` 的静态节点流验证。
- 旧版 `lab.html` 的单点 AI 验证。

本轮验收只认以下目标：

- `ScLayout` 无左侧栏。
- 主舞台顶部 3 个 KPI 卡。
- 主页运行中任务卡片。
- 右侧 rail 底部新增 Tab。
- `ScReteEditor` 编排。
- `ai-starter` + `job-starter` 真集成。

## 2. 前置条件

- [ ] `spring-support-spider-starter` 已提供 `/v1/spider/*` 新接口。
- [ ] 前端蜘蛛页已经切到最新无左栏布局。
- [ ] `ScReteEditor` 已真实挂载到任务编辑页。
- [ ] `job-starter` 可用，支持创建、暂停、恢复、删除 job。
- [ ] `ai-starter` 可用，支持节点级建议和任务级审查。
- [ ] 已准备真实浏览器测试环境。
- [ ] 已准备凭证引用机制，不把明文账号密码写入任务定义。

## 3. 页面结构测试

### 3.1 布局

- [ ] 页面为 `ScLayout`，且 `leftEnabled=false`。
- [ ] 主舞台占据主体区域，右侧为 rail。
- [ ] 右侧 rail 底部存在“添加任务”按钮。
- [ ] 不再出现旧版左侧卡片列。

### 3.2 KPI 区

- [ ] 主舞台顶部同排展示 `定时器数量`、`最新执行任务`、`任务总数`。
- [ ] `定时器数量` 来自真实 job 绑定聚合数据。
- [ ] `最新执行任务` 展示真实任务名称和最近执行时间。
- [ ] `任务总数` 不包含未保存脏草稿。
- [ ] 接口失败时 KPI 区有单独降级态。

### 3.3 主页卡片

- [ ] 默认进入 `HOME` 主页。
- [ ] 主页展示运行中的爬虫任务卡片。
- [ ] 卡片展示任务名、状态、执行类型、最近执行时间。
- [ ] 卡片展示 Job 绑定态、AI 启用态、成功/失败计数。
- [ ] 卡片支持打开、运行、暂停、日志、复制。
- [ ] 无数据时显示空态，不残留旧卡片。

## 4. rail 与 Tab 测试

### 4.1 rail 行为

- [ ] rail 中存在固定 `HOME` Tab。
- [ ] `HOME` Tab 不可关闭。
- [ ] rail 底部点击“添加任务”后创建新的草稿 Tab。
- [ ] 新创建的 Tab 会自动变成当前激活项。

### 4.2 Tab 生命周期

- [ ] 可连续创建多个任务 Tab。
- [ ] 切换 Tab 时，当前表单和编辑器状态不丢失。
- [ ] 未保存 Tab 关闭时弹出确认。
- [ ] 关闭一个 Tab 不影响其他 Tab。
- [ ] 任务名称修改后，rail 中标题自动更新。

## 5. 任务表单测试

### 5.1 基础信息

- [ ] 支持填写任务名称、任务编码、入口 URL、任务说明、标签。
- [ ] 支持配置认证方式或凭证引用。
- [ ] 禁止直接持久化明文账号密码。

### 5.2 执行类型

- [ ] 支持 `ONCE`。
- [ ] 支持 `REPEAT_N`。
- [ ] 支持 `SCHEDULED`。
- [ ] 切换执行类型后互斥字段会被清理。

### 5.3 `ONCE`

- [ ] 不展示 cron 和 job 绑定字段。
- [ ] 手动运行后立即进入执行态。

### 5.4 `REPEAT_N`

- [ ] 支持次数、间隔、并发、失败重试配置。
- [ ] 达到设定次数后自动结束。

### 5.5 `SCHEDULED`

- [ ] 支持 cron、时区、misfire 策略、job 通道配置。
- [ ] 保存后会创建真实 `job-starter` 任务。
- [ ] 修改后会同步更新已绑定 job。

## 6. `ScReteEditor` 编排测试

### 6.1 初始状态

- [ ] 新建任务后自动加载 `START` 和 `END` 节点。
- [ ] `START` 和 `END` 不可删除。
- [ ] `START` 和 `END` 不能被改成普通节点。

### 6.2 节点类型

- [ ] 可添加 `DOWNLOADER`。
- [ ] 可添加 `PARSER`。
- [ ] 可添加 `FILTER`。
- [ ] 可添加 `PIPELINE`。
- [ ] 可添加 `ERROR_HANDLER`。

### 6.3 连接和校验

- [ ] 节点之间连接为有向边。
- [ ] 不允许悬空节点。
- [ ] 不允许非法回环。
- [ ] 校验时能指出缺失链路、断链、无结束路径等问题。

### 6.4 属性联动

- [ ] 选中节点后属性面板显示正确节点数据。
- [ ] 修改节点名称、输入输出、说明后能同步回写。
- [ ] 节点删除或新增后，`ScReteEditor` 数据与表单同步。

## 7. AI 大脑测试

### 7.1 状态与降级

- [ ] `GET /v1/spider/ai/status` 能返回真实 AI 状态。
- [ ] AI 不可用时展示统一降级说明。
- [ ] AI 不可用不影响人工继续配置任务。

### 7.2 节点级 AI

- [ ] 下载器节点支持请求头、反爬、动态渲染、重试建议。
- [ ] 解析器节点支持 XPath/CSS/字段映射建议。
- [ ] 过滤器节点支持脏数据和异常样本建议。
- [ ] 输出节点支持字段类型和 DDL 建议。
- [ ] AI 建议可选择“应用到当前节点”。

### 7.3 任务级 AI

- [ ] AI 大脑支持对整条编排进行完整性审查。
- [ ] AI 能指出高风险节点、缺失节点和脆弱规则。
- [ ] AI 响应可缓存并在 Tab 切换后恢复查看。

## 8. 运行时与 `job-starter` 测试

### 8.1 `ONCE`

- [ ] 手动触发一次执行。
- [ ] 不创建长期 job。
- [ ] 运行结果写回运行状态和日志。

### 8.2 `REPEAT_N`

- [ ] 按设置次数执行。
- [ ] 中间失败时按策略重试或终止。
- [ ] 执行完毕后状态收敛正确。

### 8.3 `SCHEDULED`

- [ ] 创建时注册 job。
- [ ] 暂停任务时暂停 job。
- [ ] 恢复任务时恢复 job。
- [ ] 删除任务时解绑并删除 job。
- [ ] job 执行结果回写 `SpiderRuntimeSnapshot`。

## 9. 真实浏览器样例任务测试

### 9.1 Gitee 全部项目分页持续爬虫

- [ ] 通过真实浏览器创建任务，不通过手工拼 JSON。
- [ ] 入口地址为 `https://gitee.com/explore/all?order=starred&page=1`。
- [ ] 任务类型为 `SCHEDULED`。
- [ ] 编排至少包含：`START -> DOWNLOADER -> PARSER -> FILTER -> PIPELINE -> END`。
- [ ] 下载器节点支持浏览器渲染站点。
- [ ] 解析器节点能提取项目名称、仓库地址、分页信息。
- [ ] 能识别并继续翻页。
- [ ] 连续执行多轮后运行状态稳定。

### 9.2 百度网盘当前设备树爬虫

- [ ] 通过真实浏览器创建任务，不通过手工拼 JSON。
- [ ] 入口地址为 `https://pan.baidu.com`。
- [ ] 任务类型为 `ONCE`。
- [ ] 编排至少包含：`START -> DOWNLOADER/BROWSER_AUTH -> PARSER -> PIPELINE -> END`。
- [ ] 登录态通过凭证引用或会话引用提供。
- [ ] 不把明文账号密码写入任务定义或日志。
- [ ] 能解析当前设备树形结构。
