# 爬虫任务清单

## 1. 基线重置

本清单以 2026-04-14 最新确认需求为唯一基线，旧版“左列首页 + 右列工作台”的方案全部作废。

本轮固定要求如下：

- 页面必须基于 `ScLayout`，且 `leftEnabled=false`，不再保留左侧栏。
- 中间主舞台顶部必须同级展示 3 个指标卡：`定时器数量`、`最新执行任务`、`任务总数`。
- 中间主页默认展示“运行中的爬虫任务卡片”，不再是旧表格，也不是左侧卡片列。
- 最右侧 rail 底部通过 `ScLayout` 的 `rail-footer` slot 放置“添加”按钮。
- 点击“添加”后创建新的任务 Tab，Tab 内即任务创建/编辑工作台。
- 任务必须支持执行类型切换：`一次`、`N次`、`定时`。
- 任务编排必须使用 `ScReteEditor`，而不是自绘静态节点流。
- 编排必须强制包含 `START` 和 `END` 系统节点。
- 下载器、解析器、过滤器、输出器等节点都要能接入 `ai-starter`，形成“AI 大脑”能力。
- 定时任务必须真实接入 `job-starter`，不能只保存一个 cron 字符串。

## 3. P0 必做任务

### 3.1 文档与产品模型

- [x] 新建并维护 [`爬虫设计文档.md`](./爬虫设计文档.md)，作为唯一设计基线。
- [x] 明确页面壳模型：`ScLayout(main + rail)`，无左侧栏。
- [x] 明确主页模型：主舞台展示指标条 + 运行任务卡片区。
- [x] 明确工作台模型：右侧 rail 负责切换，主舞台负责承载当前 Tab 内容。
- [x] 明确任务调度模型：`ONCE` / `REPEAT_N` / `SCHEDULED`。
- [x] 明确编排模型：`ScReteEditor + START + END + 有向边 + AI 大脑`。

### 3.2 前端壳与首页

- [ ] `SpiderWorkbenchPage.vue` 重构为 `ScLayout` 无左栏版本。
- [ ] 主舞台顶部固定 3 个 KPI 卡，且在桌面端同排显示。
- [ ] KPI 卡语义固定：
- [ ] `定时器数量` = 已成功绑定到 `job-starter` 的任务数。
- [ ] `最新执行任务` = 最近一次开始或完成执行的任务摘要。
- [ ] `任务总数` = 已保存任务总数，不含临时脏草稿。
- [ ] 首页运行任务卡片区支持卡片网格或卡片流，不再出现旧表格。
- [ ] 运行任务卡片必须展示：
- [ ] 任务名称、运行状态、执行类型、最近执行时间。
- [ ] Job 绑定态、AI 启用态、成功/失败计数。
- [ ] 快捷动作：打开、运行、暂停、查看日志、复制。
- [ ] 首页空状态、加载中、接口失败三种状态独立处理。

### 3.3 右侧 rail 与 Tab 生命周期

- [ ] 右侧 rail 保留一个固定 `HOME` Tab，用于返回主页。
- [ ] `rail-footer` 底部提供唯一“添加任务”按钮。
- [ ] 点击“添加任务”后创建新的 `DRAFT` Tab，并自动切换到该 Tab。
- [ ] Tab 类型至少支持：
- [ ] `HOME`：主页，不可关闭。
- [ ] `DRAFT`：新建草稿。
- [ ] `TASK`：已存在任务的编辑页。
- [ ] Tab 关闭前检查未保存状态并弹出确认。
- [ ] Tab 切换时保留 `ScReteEditor` 数据、表单状态、AI 建议缓存。
- [ ] Tab 标题支持根据任务名称自动回写。

### 3.4 任务创建/编辑工作台

- [ ] 新建任务 Tab 内必须包含以下基础字段：
- [ ] 任务名称。
- [ ] 任务编码。
- [ ] 入口 URL / Seed。
- [ ] 任务说明。
- [ ] 标签 / 场景分类。
- [ ] 认证方式或凭证引用。
- [ ] 执行类型切换区必须支持：
- [ ] 一次：手动执行一次。
- [ ] N次：配置次数、间隔、并发、失败重试。
- [ ] 定时：配置 cron、时区、misfire 策略、`job-starter` 通道。
- [ ] 不同执行类型切换时，互斥字段必须自动清理。
- [ ] 不允许把明文账号密码直接写入任务 JSON；必须改为凭证引用或会话引用。

### 3.5 `ScReteEditor` 编排集成

- [ ] 工作台主编辑区域改为 `ScReteEditor`。
- [ ] `ScReteEditor` 初始加载时自动注入 `START` 和 `END` 系统节点。
- [ ] `START` 和 `END` 不可删除、不可改类型、不可脱离主链路。
- [ ] 预置节点类型至少包含：
- [ ] `DOWNLOADER`
- [ ] `PARSER`
- [ ] `FILTER`
- [ ] `PIPELINE`
- [ ] `ERROR_HANDLER`
- [ ] 编排校验必须检查：
- [ ] 开始/结束存在。
- [ ] 主链路可达。
- [ ] 不存在悬空节点。
- [ ] 不存在非法回环。
- [ ] 错误处理链路合法。
- [ ] 节点属性、输入输出、AI 配置必须和 `ScReteEditor` 当前选中节点联动。

### 3.6 AI 大脑

- [ ] 新增任务级 `AI Brain` 区域，管理模型、策略、上下文窗口、开关。
- [ ] 节点级 AI 能力最少覆盖：
- [ ] 下载器：请求头建议、反爬建议、动态渲染建议、失败重试建议。
- [ ] 解析器：XPath/CSS/字段映射建议、结构化结果建议。
- [ ] 过滤器：脏数据诊断、去重规则建议、异常样本识别。
- [ ] 输出器：字段类型推断、DDL 建议、目标介质建议。
- [ ] AI 面板要支持：
- [ ] 获取建议。
- [ ] 应用建议到当前节点。
- [ ] 查看最近一次 AI 响应。
- [ ] AI 不可用时展示统一降级状态，不阻塞人工配置。

### 3.8 `job-starter` 集成

- [ ] `ONCE` 模式不注册 job，仅即时执行。
- [ ] `REPEAT_N` 模式由蜘蛛运行时内部控制，不落长期 job。
- [ ] `SCHEDULED` 模式必须创建真实 job，并回写 `SpiderJobBinding`。
- [ ] 修改 cron、暂停、恢复、删除任务时同步变更 job。
- [ ] job 回调结果必须回写 `SpiderRuntimeSnapshot` 和 `SpiderExecutionRecord`。

- [ ] 样例任务 1：`Gitee 全部项目分页持续爬虫`

- [ ] 模式：`SCHEDULED`
- [ ] 目标：验证分页、持续运行、反爬、无主账号限制的公共页面采集。
- [ ] 样例任务 2：`百度网盘当前设备树爬虫`
- [ ] 入口：`https://pan.baidu.com`
- [ ] 模式：`ONCE`
- [ ] 目标：验证登录态、真实浏览器、树形结构解析、敏感凭证引用。
- [ ] 两个样例任务都必须通过真实浏览器方式创建，不允许只写假 JSON。
