# 爬虫设计文档

## 1. 文档目的

本文档用于把爬虫平台的产品模型、页面结构、后端边界、运行模型和验收口径一次拉齐，作为 `spring-support-spider-starter` 后续实现的唯一设计基线。

本轮最重要的结论只有一句：

`爬虫平台不是旧版任务列表页面的补丁，而是一个以 ScLayout + ScReteEditor + ai-starter + job-starter 为核心的高定制化蜘蛛工作台。`

## 2. 需求冻结

### 2.1 已确认的强约束

1. 页面必须基于 `ScLayout`。
2. 页面不能再有左侧栏，`leftEnabled=false`。
3. 中间主舞台顶部必须同级展示：
   - `定时器数量`
   - `最新执行任务`
   - `任务总数`
4. 主页就是“运行中的爬虫任务卡片”。
5. 最右侧 rail 的底部必须有“添加任务”按钮，按钮来自 `ScLayout` 的 `rail-footer` slot。
6. 点击“添加任务”后创建新的任务 Tab，Tab 内容就是任务创建/编辑工作台。
7. 任务必须支持 3 种执行类型：
   - `ONCE`
   - `REPEAT_N`
   - `SCHEDULED`
8. 任务编排必须使用 `ScReteEditor`。
9. 所有编排必须强制包含 `START` 和 `END`。
10. 下载器、解析器、过滤器、输出器等节点都支持 AI 大脑。
11. 定时任务必须真实接入 `job-starter`。

### 2.2 旧方案废弃项

以下方案不再采用：

- 旧版“左列任务卡片、右列编辑区”的页面分区。
- 靠静态按钮或自定义伪流程图模拟编排。
- 把 AI 单独放在实验页而不是集成到节点里。
- 只用 `spiderTaskScheduleType + spiderTaskCron` 表示所有调度语义。
- 把登录账号密码直接塞到任务 JSON 里。

## 3. 产品定位

`spring-support-spider-starter` 的目标定位是 `Spider Platform Starter`，不再只是默认大脑注入组件。

它最终应该同时承载：

- 蜘蛛控制台页面和静态资源。
- 任务定义、编排定义、运行状态、执行记录。
- `ai-starter` 的节点级和任务级 AI 能力。
- `job-starter` 的调度绑定和结果回写。
- 可被真实浏览器驱动的创建、编辑、运行、验证链路。

## 4. 页面信息架构

## 4.1 Shell 结构

页面壳固定为：

- 主组件：`ScLayout`
- `leftEnabled=false`
- `railEnabled=true`
- 中间 `main`：主舞台
- 右侧 `rail`：Tab 导航和新增任务入口

建议结构如下：

- `HOME` 固定 Tab：返回主页，不可关闭。
- `TASK-*` 动态 Tab：任务草稿或任务编辑页，可关闭。
- `rail-footer`：唯一“添加任务”按钮。

这样既满足“没有左侧栏”，又保留主页和任务编辑页之间的快速切换能力。

## 4.2 主舞台两种状态

### `HOME` 状态

显示内容：

- 顶部 KPI 条
- 运行中的爬虫任务卡片
- 最新执行摘要
- 异常或告警摘要

### `TASK-*` 状态

显示内容：

- 顶部 KPI 条
- 当前任务的基础信息表单
- 执行类型配置区
- `ScReteEditor` 编排画布
- 节点属性区
- AI 大脑区
- 运行日志 / 运行快照区

## 4.3 顶部 KPI 条

KPI 固定为 3 个卡片，同排展示：

### 1. 定时器数量

定义：

- 当前已绑定到 `job-starter` 的有效调度任务数。

### 2. 最新执行任务

定义：

- 最近一次开始执行或完成执行的任务摘要。

显示建议：

- 任务名称
- 状态
- 时间

### 3. 任务总数

定义：

- 当前已保存任务总数，不含未持久化的脏草稿。

## 4.4 主页卡片区

主页卡片只展示“运行中”或“需要运营关注”的任务，不展示纯历史列表。

每张卡片至少包含：

- 任务名称
- 当前状态
- 执行类型
- 最近执行时间
- Job 绑定态
- AI 启用态
- 成功/失败计数
- 快捷操作：打开、运行、暂停、日志、复制

## 5. 任务工作台设计

## 5.1 Tab 创建流程

1. 点击 `rail-footer` 底部“添加任务”按钮。
2. 系统创建一个新的 `DRAFT` Tab。
3. Tab 自动激活，主舞台进入任务编辑态。
4. 默认注入空白任务元信息和默认编排。
5. 默认编排中自动加入 `START` 和 `END`。

## 5.2 任务基础信息

基础信息字段至少包括：

- 任务名称
- 任务编码
- 入口 URL / Seed
- 任务说明
- 任务标签 / 场景分类
- 认证方式
- 凭证引用 / 会话引用

安全规则：

- 不允许把明文账号密码写入任务持久化结构。
- 凭证必须通过 `SpiderCredentialRef` 或会话引用来关联。

## 5.3 执行类型模型

### `ONCE`

特征：

- 仅手动触发一次。
- 不注册长期 job。

字段建议：

- `manualTrigger=true`
- `threadCount`
- `retryPolicy`

### `REPEAT_N`

特征：

- 按设定次数执行。
- 到达次数后自动结束。
- 不落长期 `job-starter` 绑定。

字段建议：

- `repeatTimes`
- `repeatInterval`
- `threadCount`
- `retryPolicy`

### `SCHEDULED`

特征：

- 接入 `job-starter`。
- 必须创建真实调度任务。

字段建议：

- `cron`
- `timezone`
- `misfirePolicy`
- `jobChannel`
- `jobBindingId`

## 5.4 任务状态模型

建议任务状态至少包含：

- `DRAFT`
- `READY`
- `RUNNING`
- `PAUSED`
- `FAILED`
- `FINISHED`

## 6. `ScReteEditor` 编排模型

## 6.1 核心原则

- 编排必须由 `ScReteEditor` 承载。
- 不是静态展示图，而是真实的数据结构编辑器。
- 每次切换 Tab 后，编辑器数据要完整恢复。

## 6.2 必备系统节点

每个任务在创建时自动生成：

- `START`
- `END`

约束：

- 不可删除。
- 不可修改为其他类型。
- 必须位于主链路首尾。

## 6.3 首批节点类型

首批落地节点建议如下：

- `START`
- `DOWNLOADER`
- `PARSER`
- `FILTER`
- `PIPELINE`
- `ERROR_HANDLER`
- `END`

## 6.4 边与连线规则

- 所有连线必须是有向边。
- 主链路必须从 `START` 可达 `END`。
- 不允许悬空节点。
- 不允许非法回环。
- 允许通过 `ERROR_HANDLER` 表达失败链路，但不能破坏主链路收口。

## 6.5 画布交互

`ScReteEditor` 至少要支持：

- 节点新增
- 节点删除
- 节点连线
- 节点选中
- 节点属性编辑
- 自动排列
- 数据导入导出
- 校验结果提示

## 7. AI 大脑设计

## 7.1 设计目标

AI 不是额外页面，而是任务工作台内的原生能力。

AI 分为两层：

- 任务级 AI 大脑
- 节点级 AI 助手

## 7.2 任务级 AI 大脑

负责：

- 基于整条编排给出完整性审查。
- 指出缺失节点、脆弱节点、高风险规则。
- 聚合最近日志与失败样本做诊断。

建议字段：

- `provider`
- `model`
- `temperature`
- `contextWindow`
- `enabled`

## 7.3 节点级 AI 助手

### 下载器节点

AI 能力：

- 请求头建议
- 反爬建议
- 动态渲染建议
- 重试策略建议

### 解析器节点

AI 能力：

- XPath/CSS 建议
- 字段映射建议
- 结构化结果建议

### 过滤器节点

AI 能力：

- 脏数据诊断
- 去重规则建议
- 异常样本建议

### 输出节点

AI 能力：

- 字段类型推断
- DDL 建议
- 输出介质建议

## 7.4 降级策略

AI 不可用时：

- 页面必须给出清晰降级提示。
- 节点仍然可以人工配置。
- 不允许因为 AI 失败导致整页不可用。

## 8. 后端架构设计

## 8.1 模块边界

`spring-support-spider-starter` 需要承载：

- 页面资源与路由接入
- Dashboard 接口
- Workbench 接口
- Task 接口
- Flow 接口
- AI 接口
- Runtime 接口

`job-starter` 负责：

- 定时任务注册
- 暂停 / 恢复 / 删除
- 调度执行结果回写

`ai-starter` 负责：

- 节点建议
- 任务审查
- AI 诊断

## 8.2 领域对象

建议领域对象如下：

- `SpiderTaskDefinition`
- `SpiderExecutionPolicy`
- `SpiderFlowDefinition`
- `SpiderFlowNode`
- `SpiderFlowEdge`
- `SpiderAiProfile`
- `SpiderCredentialRef`
- `SpiderJobBinding`
- `SpiderRuntimeSnapshot`
- `SpiderExecutionRecord`

## 8.3 接口设计

### Dashboard

- `GET /v1/spider/dashboard/summary`
- `GET /v1/spider/dashboard/running-cards`

### Workbench

- `GET /v1/spider/workbench/tabs`
- `POST /v1/spider/workbench/tabs`
- `PUT /v1/spider/workbench/tabs/{tabId}`
- `DELETE /v1/spider/workbench/tabs/{tabId}`

### Task

- `POST /v1/spider/tasks`
- `GET /v1/spider/tasks/{taskId}`
- `PUT /v1/spider/tasks/{taskId}`
- `DELETE /v1/spider/tasks/{taskId}`

### Flow

- `GET /v1/spider/tasks/{taskId}/flow`
- `PUT /v1/spider/tasks/{taskId}/flow`
- `POST /v1/spider/tasks/{taskId}/flow/validate`

### AI

- `GET /v1/spider/ai/status`
- `POST /v1/spider/tasks/{taskId}/ai/review`
- `POST /v1/spider/tasks/{taskId}/nodes/{nodeId}/ai/suggest`
- `POST /v1/spider/tasks/{taskId}/nodes/{nodeId}/ai/apply`

### Runtime

- `POST /v1/spider/tasks/{taskId}/run`
- `POST /v1/spider/tasks/{taskId}/pause`
- `POST /v1/spider/tasks/{taskId}/resume`
- `GET /v1/spider/tasks/{taskId}/runtime`
- `GET /v1/spider/tasks/{taskId}/records`

## 9. 运行时设计

## 9.1 执行入口

- `ONCE`：用户手动触发。
- `REPEAT_N`：运行时内部控制循环。
- `SCHEDULED`：由 `job-starter` 回调触发。

## 9.2 执行链路

执行器按主链路推进：

`START -> DOWNLOADER -> PARSER -> FILTER -> PIPELINE -> END`

需要支持：

- 上游输出传给下游输入
- 节点级日志
- 节点级失败收敛
- 整体运行快照

## 9.3 持久化回写

执行过程中需要回写：

- 当前状态
- 最近执行时间
- 成功/失败计数
- 最近错误摘要
- Job 绑定状态
- 节点日志摘要

## 10. 样例任务模板

## 10.1 Gitee 全部项目分页持续爬虫

目标：

- 验证公共站点分页采集、持续调度、真实浏览器渲染页面解析。

建议配置：

- 入口：`https://gitee.com/explore/all?order=starred&page=1`
- 执行类型：`SCHEDULED`
- 编排：`START -> DOWNLOADER -> PARSER -> FILTER -> PIPELINE -> END`
- 下载器：浏览器下载 + 反爬建议
- 解析器：提取项目名称、地址、分页信息
- 过滤器：去重
- 输出：任务明细或归档表

## 10.2 百度网盘当前设备树爬虫

目标：

- 验证登录态站点、真实浏览器驱动、树形结构提取、凭证安全策略。

建议配置：

- 入口：`https://pan.baidu.com`
- 执行类型：`ONCE`
- 认证：凭证引用或会话引用
- 编排：`START -> DOWNLOADER/BROWSER_AUTH -> PARSER -> PIPELINE -> END`
- 输出：设备树形结构

安全要求：

- 凭证不落明文。
- 日志不打印明文密码。

## 11. 完成标准

以下全部满足，才算本轮设计被正确实现：

- 没有左侧栏。
- 主舞台顶部稳定展示 3 个 KPI 卡。
- 主页能展示运行中的爬虫任务卡片。
- 右侧 rail 底部“添加任务”按钮可持续创建新 Tab。
- 新 Tab 内可完成任务信息、调度类型、`ScReteEditor` 编排、AI 大脑配置。
- `START` / `END` 强约束有效。
- `ONCE` / `REPEAT_N` / `SCHEDULED` 三种模式真实可用。
- `SCHEDULED` 和 `job-starter` 已打通。
- 下载器、解析器、过滤器、输出器节点能调用 `ai-starter`。
- 真实浏览器已成功创建 Gitee 与百度网盘两个样例任务。
